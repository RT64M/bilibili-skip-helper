# Bilibili Skip Helper

基于 B 站弹幕的时间线索，自动识别并跳过 UP 主恰饭片段。

一个小型 Chromium MV3 插件：在视频页读取弹幕，当多条弹幕指向同一时间点时，把这些时间线索聚合成候选广告段，在播放到该位置前提示并自动跳过。无需登录、无需服务器、无需构建。

## 安装

1. 打开 Chrome 或 Edge 的扩展管理页。
2. 开启开发者模式。
3. 选择“加载已解压的扩展程序”。
4. 选择本项目文件夹。

## 使用

打开 `https://www.bilibili.com/video/*` 视频页即可。插件会在后台读取视频信息与弹幕 XML，并在分析完成后显示 3 秒识别结果。

如果找到候选跳转点，播放到广告开始前约 2 秒时，页面右上角会显示“跳过 / 忽略”提示，并开始 5 秒倒计时。倒计时结束仍未操作时，会自动跳到候选目标时间。

如果你手动拖动进度条，或直接跳转到了识别出的广告时间段内部，插件会改用 3 秒倒计时自动跳出广告段；跳过后可短暂撤回。

当插件判断你是在主动回看广告段时，会停止本段自动跳过：包括已经在广告段内又把进度条向前回拉，或已经播放到广告结束 20 秒之后又跳回广告段内部。

弹幕线索经常会比广告真正开始稍早出现，所以插件会把识别出的广告开始时间轻微后移，减少过早弹窗和过早判定进入广告段。

## 诊断

如果某个视频没有识别出广告，可以在视频 URL 后加 `?bsh_debug=1`，刷新页面后打开浏览器开发者工具 Console。插件会输出弹幕诊断信息，包括读取到的弹幕数量、带时间码的弹幕数量、通过关键词筛选的数量、候选聚合结果和被过滤原因。

时间码支持 `5:50`、`550`、`0550` 这类写法，也支持 `伍伍零` 这类中文数字紧凑写法；这些内容可以嵌在弹幕句子中，例如 `感谢伍伍零工程`。

## 说明

- 无构建步骤，无外部依赖。
- 不上传数据，不使用自建服务器。
- 不发送弹幕，不修改账号状态。
- 只处理视频内容片段，不处理网页广告位。
- 识别效果取决于弹幕质量，可能漏判或误判。
- 如果使用“加载已解压的扩展程序”安装，更新文件后需要在扩展管理页手动重新加载插件。
